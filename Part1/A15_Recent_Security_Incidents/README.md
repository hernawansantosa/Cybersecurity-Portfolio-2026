# A15 — Discover 5 Recent Security Incidents

This activity reviews five recent cybersecurity incidents and analyses their causes, impacts, and lessons learned.

---

## 1. CrowdStrike Global IT Outage (2024)

### Description  
A faulty software update from CrowdStrike caused widespread system crashes affecting airlines, airports, and businesses globally.

### Cause  
Software update failure (not a cyberattack but a critical security-related failure).

### Impact  
- Flight disruptions worldwide  
- System outages across industries  
- Operational downtime  

### Lesson  
Even security tools can create large-scale risks if not properly tested.

---

## 2. Ticketmaster Data Breach (2024)

### Description  
Ticketmaster experienced a major data breach exposing customer information.

### Cause  
Third-party cloud database vulnerability.

### Impact  
- Customer data exposed  
- Privacy concerns  
- Reputational damage  

### Lesson  
Third-party vendors are a major cybersecurity risk.

---

## 3. Change Healthcare Ransomware Attack (2024)

### Description  
A ransomware attack disrupted healthcare systems in the United States.

### Cause  
Compromised credentials and lack of MFA.

### Impact  
- Payment systems disrupted  
- Healthcare delays  
- Financial losses  

### Lesson  
Strong authentication and MFA are critical.

---

## 4. Microsoft Email Breach (2024)

### Description  
Attackers gained access to Microsoft corporate email accounts.

### Cause  
Token-based authentication vulnerability.

### Impact  
- Sensitive communications exposed  
- Security concerns across organizations  

### Lesson  
Authentication systems must be continuously monitored and secured.

---

## 5. Australian Superannuation Cyber Incidents (2024)

### Description  
Cyber incidents targeted Australian financial systems and customer accounts.

### Cause  
Credential stuffing and phishing attacks.

### Impact  
- Unauthorized access attempts  
- Financial risk to users  

### Lesson  
Users must avoid password reuse and enable MFA.

---

## Reflection

These incidents highlight that cybersecurity risks can arise from technical vulnerabilities, human error, and third-party dependencies. Strong authentication, regular updates, and risk management are essential to prevent large-scale disruptions.
